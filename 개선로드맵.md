## 2. 확정된 결정사항

- **전과목 업로드 저장 정책 (A안)**: 전과목 모드에서만 "검증 통과 시 과목별 자동 [저장]" 허용.
  단일 과목 입력은 기존대로 저장 안 함. 시작 전 명시 동의, 검증 실패 과목은 저장 없이 중단·리포트.
- **HWP 처리 파이프라인**: pdf→Gemini 직접 / hwpx→XML 텍스트 추출 / hwp→한컴 COM으로 PDF 변환(설치 시, 보안모듈 자동등록), 미설치 시 바이너리 텍스트 추출 폴백.
- **AI 양식 인식은 범용 프롬프트 1개** — 업체별 파서를 만들지 않는다. 샘플 6종(ref/, 이지에듀·스쿨마스터 × pdf·hwp·hwpx)은 테스트 케이스.
- **키 로테이션은 DooClick 키서버 로직 이식** — APIKeys 시트(A:키, B:상태, C:마지막사용) 공유.
  마지막사용 오래된 순 선택 + 사용 시 C열 갱신 + 429 시 "한도초과" 기록 후 다음 키 재시도.
- **AI 가져오기 결과는 검토 화면에서 확인 후 적용** (바로 반영 금지 — 오인식 방지).

## 3. Phase별 계획·진행 상황

### Phase 0 — 준비 ✅
- ✅ ref/ 정리 커밋 (기존 DooClick 참고 코드 삭제 반영 + .gitignore에 ref/, *.zip 추가 — 로컬 샘플은 untracked로 보존됨)
- ✅ 샘플 자료 확보 (ref/ 6종) · code.txt(DooClick 키서버) 확인 · APIKeys 시트 컬럼 확정

### Phase 1 — 작업공간 기반 (요청 7·8·9) ⬜
모든 인앱 편집 기능의 저장 토대. 가장 먼저.
- ⬜ 작업공간 폴더 `문서\NeisAutoFill\` — 인앱 생성·편집 자료의 기본 저장 위치
- ⬜ 성적·평가계획 자동 저장 (편집 후 디바운스, 엑셀에서 열어둔 파일 잠금 대응)
- ⬜ 시작 시 최근 자료 자동 로드 (settings에 최근 경로 기록)
- ⬜ 툴바 [최근 파일] 메뉴 + 기존 열기·드래그앤드롭 유지
- 서술문 xlsx 자동 미러·재불러오기는 Phase 4에서 (생성기 개편과 함께)

### Phase 2 — 프로그램 내 편집 (요청 2·3·4·5) ⬜
- ⬜ 학생 명단 편집 창 — 엑셀 표 Ctrl+V 일괄 붙여넣기 (TSV 파싱, 번호/이름 자동 인식)
- ⬜ 성취기준(평가계획) 편집 그리드 — 과목·영역·기준 인앱 수정, 붙여넣기 지원
- ⬜ 성적표 다중 셀 선택 + 복사/붙여넣기 (척도 화이트리스트 검증)
- ⬜ 선택 셀 일괄 등급 지정 (우클릭 메뉴 + 숫자 단축키)
- ⬜ 성취기준 참조 패널 (메인 우측 접이식, 토글 상태 저장)

### Phase 3 — AI 양식 가져오기 (요청 1·9) ⬜
- ⬜ 파일 판별·추출 파이프라인 (pdf/hwpx/hwp — 위 결정사항 참조)
- ⬜ 한컴 COM 변환기 + 보안모듈 1회 자동 등록
- ⬜ GAS `parsePlan` 액션 (파일/텍스트 → 구조화 JSON, 스키마: 과목/영역/성취기준/등급별 기준)
- ⬜ 가져오기 검토 창 (AI 결과 편집 후 [적용])
- ⬜ 샘플 6종 전부 인식 검증
- ⬜ 드래그앤드롭에 pdf/hwpx/hwp 연결

### Phase 4 — 생성기 개선 (요청 6·10) ⬜
- ⬜ 백그라운드 생성 큐 (창 닫기·과목 전환에도 계속, 동시 2~3건, 진행률·취소)
- ⬜ 전과목/선택과목 일괄 생성 (과목 체크리스트)
- ⬜ 서술문 xlsx 자동 저장·불러오기 (요청 7의 서술문 부분)

### Phase 5 — 나이스 입력 대화형 매핑 (요청 11) ⬜
- ⬜ 입력 전 "매칭 미리보기 창" — 실행 시 화면을 먼저 읽고 매칭 결과 표 표시
- ⬜ 이름 불일치: 유사도 자동 제안 + 드롭다운 확정/제외
- ⬜ 과목명 불일치: 경고 배너 + 계속 여부
- ⬜ 영역명 불일치: 영역 매핑 드롭다운
- ⬜ 영역 수 불일치: 입력할 영역 선택 + 순서 지정
- ⬜ 문제 없으면 창 생략하고 바로 실행 (기존 흐름 유지)
- ⬜ StudentMatcher에 매핑 오버라이드 파라미터 (기존 테스트 유지)

### Phase 5.5 — 전과목 자동 업로드 (요청 14) ⬜
- ⬜ 나이스 메뉴·과목 콤보·[조회] 자동 조작 (셀렉터 실측 필요 — DomInspector 덤프)
- ⬜ 전과목 오케스트레이터: 과목 선택→조회→입력→전체 재검증→**통과 시 [저장]**→다음 과목 (A안)
- ⬜ 등급([교과별 평가])·서술문([학기말 종합의견]) 두 화면 모두 지원
- ⬜ 시작 전 전과목 매칭 미리보기(Phase 5 재사용) + 자동 저장 동의
- ⬜ 중간 실패 리포트 ("여기까지 저장됨, X과목부터 실패")
- ⬜ 개발문서.md 안전정책 문구 개정

### Phase 6 — GAS 로깅·API 키 관리 (요청 12) ⬜
code.gs 수정 + 재배포 필요. 다른 Phase와 독립 — 언제든 병행 가능.
- ⬜ 키 로테이션: 마지막사용 오래된 "사용가능" 키 선택 + C열 갱신
- ⬜ 429/한도초과 시 B열 "한도초과" 기록 + 다음 키 자동 재시도
- ⬜ 로그를 학생당 → 생성요청(배치)당 1건 (과목·인원수 기록)
- ⬜ startup 기록 동작 확인 (코드는 v1.2.2에 있음 — GAS 배포본이 구버전인지 확인)
- ⬜ GAS 재배포 + 프로그램 URL 갱신 확인

### Phase 7 — UI/UX 다듬기 (요청 13) ⬜
- ⬜ 성적 편집 Undo (Ctrl+Z) — 일괄 지정·붙여넣기 실수 복구
- ⬜ 작업 상태바 — 백그라운드 생성·입력 진행을 메인 하단에 상시 표시
- ⬜ 첫 실행 온보딩 (① 평가계획 → ② 명단 → ③ 성적 단계 안내)
- ⬜ 성적표 미입력 셀 강조
- ⬜ 설정 창 통합 (지역/척도/바이트제한 등)

## 4. 진행 순서

**0 → 1 → 2 → 4 → 5 → 5.5 → 3 → 6 → 7**

- 1·2가 먼저: 3(AI 가져오기) 결과물이 저장·편집될 곳이 필요
- 3은 외부 의존(샘플 검증·GAS 재배포·프롬프트 튜닝)이 커서 기반 완성 후
- 6은 독립적 — code.gs 준비된 상태이므로 틈나는 대로 병행
- 5.5는 5의 매핑 인프라 + 실기기 셀렉터 실측을 재사용
