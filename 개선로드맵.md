## 2. 확정된 결정사항

- **전과목 업로드 저장 정책 (A안)**: 전과목 모드에서만 "검증 통과 시 과목별 자동 [저장]" 허용.
  단일 과목 입력은 기존대로 저장 안 함. 시작 전 명시 동의, 검증 실패 과목은 저장 없이 중단·리포트.
- **HWP 처리 파이프라인**: pdf→Gemini 직접 / hwpx→XML 텍스트 추출 / hwp→한컴 COM으로 PDF 변환(설치 시, 보안모듈 자동등록), 미설치 시 바이너리 텍스트 추출 폴백.
- **AI 양식 인식은 범용 프롬프트 1개** — 업체별 파서를 만들지 않는다. 샘플 6종(ref/, 이지에듀·스쿨마스터 × pdf·hwp·hwpx)은 테스트 케이스.
- **키 로테이션은 DooClick 키서버 로직 이식** — APIKeys 시트(A:키, B:상태, C:마지막사용) 공유.
  마지막사용 오래된 순 선택 + 사용 시 C열 갱신 + 429 시 "한도초과" 기록 후 다음 키 재시도.
- **AI 가져오기 결과는 검토 화면에서 확인 후 적용** (바로 반영 금지 — 오인식 방지).

### Phase 1 — 작업공간 기반 (요청 7·8·9) ✅

- ✅ 성적 자동 저장 — 편집 2초 후 디바운스 저장(`MainViewModel.AutoSaveGrades`), 파일 잠금 시 로그만 남기고 dirty 유지, 종료 시 조용히 재시도 후 실패 때만 질문
- ✅ 시작 시 최근 자료 자동 로드 — `AppStateStore`(state.json)에 마지막 평가계획서·성적파일 기록, 시작 시 복원 (실패는 로그만)
- ✅ 툴바 [🕘 최근 ▾] 메뉴 — 최근 10개(계획서/성적 구분), 실존 파일만 표시. 기존 열기·드래그앤드롭 유지

- 평가계획 인앱 편집분 자동 저장은 Phase 2에서 (편집 기능과 함께)

### Phase 2 — 프로그램 내 편집 (요청 2·3·4·5) ✅
- ✅ [📝 명단·계획] 편집 창(`PlanEditorWindow`) — 학생 명단: 엑셀 복사 → 한 번에 붙여넣기(전체 교체, 번호/이름 자동 인식·헤더 스킵·이름만 복사 시 순번 자동), 행 추가/삭제
- ✅ 평가계획 편집 그리드 — 행=영역, 열=영역|성취기준|등급별 기준내용. 과목 추가/삭제/이름 변경, 엑셀 표 Ctrl+V. 저장 시 `PlanWorkbookWriter`로 작업공간 `평가계획서.xlsx`에 기록 후 재로드 (엑셀 직접 수정과 동일 경로 — 요청 7 충족)
- ✅ 성적표 다중 셀 선택(`SelectionUnit=CellOrRowHeader`) + Ctrl+C/Ctrl+V — 등급 셀은 척도 화이트리스트 검증, 허용외 값은 건너뛰고 로그
- ✅ 선택 셀 일괄 등급 지정 — 우클릭 메뉴(척도 변경 대응 동적 구성) + 숫자 1~9 단축키 + Del 지우기
- ✅ 성취기준 참조 패널 — 각 과목 탭 [📖 성취기준] 토글, 우측 300px 패널에 영역·성취기준·등급별 기준 표시, 토글 상태 저장(state.json)
- 신규 공용 코드: `ClipboardTable`(Core, TSV 파싱·명단 해석), `PlanWorkbookWriter`(Excel, 로더와 round-trip 보장), `DataGridClipboard`(App, 표 붙여넣기) — 테스트 8개 추가

### Phase 3 — AI 양식 가져오기 (요청 1·9) ✅ (GAS 재배포 후 실검증 필요)
- ✅ 파일 판별·추출 파이프라인 (`PlanFileExtractor`) — pdf=원문 base64 / hwpx=zip·XML 텍스트(표는 탭·줄바꿈 평탄화) / hwp=한컴 COM→PDF 변환 (미설치 시 안내)
- ✅ 한컴 COM 변환기 (STA 스레드 실행, RegisterModule 시도 — 보안모듈 미등록 환경은 최초 1회 승인 창 가능)
- ✅ GAS `parsePlan` 액션 — 범용 프롬프트 1개 (양식 무관), responseMimeType=JSON, 척도 라벨 동적 매핑
- ✅ 가져오기 검토 = 명단·계획 편집 창 재사용 — [📄 평가계획 불러오기] → AI 결과가 표에 채워짐 → 확인·수정 후 [저장 후 적용]
- ✅ 드래그앤드롭 pdf/hwpx/hwp → 편집 창 자동 오픈 + 가져오기
- ✅ `GasPlanImporter` 응답 매핑 (허용외 등급 키 폐기, 테스트 4개 — hwpx 실샘플 추출 포함)
- ✅ **실검증 (PDF)** — 이지에듀·스쿨마스터 PDF 인식 확인 (2단계 파싱: listPlanSubjects → 과목별 parsePlan, gemini-3.1-flash-lite). 사용자 확인 완료
- ⬜ hwp/hwpx 경로 실검증 (한컴 COM 변환 포함) — 필요 시 추후

### Phase 4 — 생성기 개선 (요청 6·10) ✅
- ✅ 백그라운드 생성 큐 (`GenerationQueue`) — 동시 2건, 창 닫기·과목 전환에도 계속, 완료 즉시 narratives.json 저장, [⛔ 생성 중지], 진행바+상태(“생성 12/48 — 창을 닫아도 계속…”)
- ✅ 전과목/선택과목 일괄 생성 — [🌐 전과목 생성] → 과목 체크리스트 대화상자(생성 가능 인원 표시) → 선택 과목 전체 큐 투입
- ✅ 서술문 xlsx 저장·불러오기 — [💾 저장]=작업공간 `서술문.xlsx` (사용자 피드백으로 자동 미러 → 수동 저장 전환), [📤 내보내기]=위치 선택, [📂 불러오기]=엑셀 수정분 되가져오기 (`NarrativeWorkbookLoader`, round-trip 테스트 2개). 툴바 기능별 2줄 재배치 (1줄 생성 / 2줄 자료·입력)

### Phase 5 — 나이스 입력 대화형 매핑 (요청 11) ✅ (실기기 검증 남음)
- ✅ 입력 전 "매칭 미리보기 창"(`MatchPreviewWindow`) — 엔진이 화면(행지도)을 읽은 뒤 `resolveMatch` 콜백으로 UI 가 검토·결정. 문제 없으면 창 생략하고 바로 실행
- ✅ 과목명 불일치: 경고 배너 + "이대로 입력 동의" 체크 필수
- ✅ 학생 이름 불일치: 유사도 자동 제안(포함/편집거리 1) + 드롭다운으로 엑셀 학생 확정 or 입력 안 함
- ✅ 영역명 불일치: 영역 매핑 드롭다운 (자동 추천 포함)
- ✅ 영역 수 불일치/중복: 순서 매핑 — 화면 행 순서별로 입력할 엑셀 영역 선택('입력 안 함'=건너뜀), 같은 영역 중복 매핑 방지 검증. 화면 영역명 중복 시 순서 모드 강제
- ✅ `StudentMatcher` areaMap/nameMap 오버라이드 + ByOrder 빈 위치 건너뜀, `MatchAnalyzer` 신설 — 테스트 6개 추가 (71/71)
- ⬜ **실기기 검증** — 실제 나이스 화면에서 불일치 시나리오 확인 (Phase 5.5 실측과 함께)

### Phase 5.5 — 전과목 자동 업로드 (요청 14) ✅ (코드 — 실기기 셀렉터 확정 필수)
- ✅ 엔진 `SelectSubjectAsync`(과목 콤보 열기→선택→[조회]→갱신 폴링) + `SaveScreenAsync`([저장]→확인/완료 대화상자 처리). 버튼은 접근성 이름(GetByRole '조회'/'저장') + 수동 스캔 폴백 — **셀렉터 잠정** (`NeisSelectors` Phase 5.5 구획)
- ✅ 메인 [🚀 전과목 입력] — 동의 대화상자(A안 명시) → 과목별 전환→입력→검증 통과 시 저장→다음. 실패/전환불가/사용자취소 시 그 자리에서 중단, 과목별 결과 요약 팝업
- ✅ 생성기 [🚀 전과목 입력] — 서술문 동일 흐름 ([학기말 종합의견] 화면에서)
- ✅ 매칭 미리보기(Phase 5) 과목마다 재사용, 개발문서 안전정책 A안 개정
- ✅ [🔎 진단] 버튼 복원 (셀렉터 실측용 DomInspector)
- ⬜ **실기기 검증 (필수)**: [교과별 평가]·[학기말 종합의견] 화면에서 [🔎 진단] 덤프 → [조회]/[저장]/대화상자 셀렉터 확정 → 소규모(1과목→2과목) 시험 후 전과목. 메뉴 간 이동(교과별평가↔종합의견) 자동화는 실측 후 추가 예정 — 현재는 해당 화면을 띄운 상태에서 시작

## 4. 진행 순서

**0 → 1 → 2 → 4 → 5 → 5.5 → 3 → 6 → 7**

- 1·2가 먼저: 3(AI 가져오기) 결과물이 저장·편집될 곳이 필요

- 6은 독립적 — code.gs 준비된 상태이므로 틈나는 대로 병행
- 5.5는 5의 매핑 인프라 + 실기기 셀렉터 실측을 재사용
