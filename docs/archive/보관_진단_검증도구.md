# 보관: 화면 진단 · 검증(dry-run) 도구

작성일: 2026-07-07
상태: **UI 에서 제거됨 (코드는 전부 유지)** — 나중에 나이스 DOM 이 바뀌거나 새 화면을
자동화할 때 이 문서대로 버튼만 다시 붙이면 즉시 사용 가능하다.

---

## 1. 화면 진단 (DomInspector)

### 무엇인가
현재 나이스 화면의 구조를 읽기 전용으로 덤프하는 도구.
- 보이는 그리드의 열/행 수, 각 셀의 aria-label·내부 컨트롤(textarea/input/combo)
- 보이는 콤보박스 라벨 목록
- 텍스트 편집기 후보 (CLX `div.cl-textarea` 등)
- **스크롤 경계 분석**: 프록시 스크롤바 바닥 이동 → 렌더된 행 상세 덤프 → CLX reveal 전후 비교
- **CLX 내부 API 탐사**: `cpr.core.Platform.INSTANCE.getAllRunningAppInstances()` 로
  그리드 컨트롤을 찾아 스크롤/행 메서드 목록 추출

### 실적 (이 도구로 해결한 것)
1. **서술문 화면 셀렉터 확정** — 편집란이 네이티브 textarea 가 아닌 `div.cl-textarea`
   (클릭 시 편집 모드 진입) 임을 발견 → NarrativeWriter 구현 근거
2. **§8 "마지막 행 미렌더링" 진짜 원인 발견** — 행은 항상 렌더되고 있었고,
   마지막 행만 aria-label 에 `"마지막 행"` 토큰이 추가되어 정규식이 못 읽던 것
3. CLX 공식 API `reveal(rowIndex)` 발견 → ClxGridApi 폴백 경로

### 코드 위치 (전부 유지됨)
- `src/NeisAutoFill.Automation/DomInspector.cs` — 도구 본체
- `INeisEngine.InspectDomAsync()` / `NeisEngine.InspectDomAsync()` — 엔진 진입점

### 다시 켜는 법
`MainWindow.xaml` 작업 줄에 버튼 추가:
```xml
<Button Content="🔎 화면 진단" Command="{Binding InspectCommand}" Padding="10,6"/>
```
`MainViewModel`에 커맨드·핸들러 추가 (제거 전 원본):
```csharp
public ICommand InspectCommand { get; }
// 생성자: InspectCommand = new AsyncRelayCommand(InspectAsync);

private async Task InspectAsync()
{
    if (!_engine.Connected) { ShowError("② 브라우저 연결을 먼저 해주세요."); return; }
    try
    {
        var report = await _engine.InspectDomAsync();
        Log(report);
        AppPaths.EnsureRoot();
        var file = Path.Combine(AppPaths.Root, $"dom_inspect_{DateTime.Now:yyyyMMdd_HHmmss}.txt");
        File.WriteAllText(file, report);
        Log($"진단 리포트 저장: {file}");
    }
    catch (Exception ex) { Log($"진단 오류: {ex.Message}"); }
}
```

### 사용 방법
1. 나이스에서 조사할 화면을 띄우고 (그리드가 보이는 상태) [② 브라우저 연결]
2. [🔎 화면 진단] 클릭 — 아무것도 조작하지 않음 (스크롤 1회 왕복만)
3. 로그와 `%AppData%\NeisAutoFill\dom_inspect_*.txt` 에서 결과 확인
4. 새 화면 자동화 시: 그리드 colcount / 셀 aria-label 형식 / 편집 컨트롤 종류를
   이 리포트에서 읽어 셀렉터·정규식을 작성한다

## 2. 검증 (dry-run)

### 무엇인가
실제 입력 없이 "화면 파싱 → 매칭 → 입력 계획"까지만 수행하고 계획을 로그로 출력.
새 화면·새 학급·새 척도에서 실입력 전 안전 확인용.

### 코드 위치 (전부 유지됨)
- 등급: `INeisEngine.RunSubjectAsync(..., bool dryRun, ...)` — dryRun=true 로 호출하면 계획만 출력
- 서술문: `INeisEngine.RunNarrativesAsync(..., bool dryRun, ...)` — 동일
- `MainViewModel.RunSubjectAsync(sheet, dryRun)` / `GeneratorViewModel.UploadAsync(dryRun)` 도
  bool 파라미터 유지 — UI 버튼만 제거됨

### 다시 켜는 법
과목 탭 (`MainWindow.xaml` SubjectTabView 템플릿):
```xml
<Button Content="검증 (dry-run)" Command="{Binding DryRunCommand}" Padding="8,5" Margin="0,0,6,0"/>
```
`SubjectViewModel` 생성자:
```csharp
DryRunCommand = new AsyncRelayCommand(() => _main.RunSubjectAsync(Sheet, dryRun: true));
public ICommand DryRunCommand { get; }
```
서술문 생성기 (`GeneratorWindow.xaml` 상단 바):
```xml
<Button Content="검증 (dry-run)" Command="{Binding UploadDryRunCommand}" Padding="10,6"/>
```
`GeneratorViewModel` 생성자:
```csharp
UploadDryRunCommand = new AsyncRelayCommand(() => UploadAsync(dryRun: true));
public ICommand UploadDryRunCommand { get; }
```

### 사용 방법
실입력과 동일한 준비(연결·파일 로드) 후 검증 버튼 클릭 → 로그에서
`행 파악 N/M` (전 행 파악 여부), `[계획 i/total]` (매칭 정확성), 건너뜀 사유를 확인.
전부 정상일 때만 실입력 진행.

## 3. 주의사항 (재사용 시)

- 진단의 CLX 탐사는 `getAllRunningAppInstances` 를 쓴다 (`getAppInstances` 아님)
- 나이스 DOM 관찰은 개발자도구로 하면 안 되는 것이 있다 — 콤보 옵션 팝업은
  포커스를 잃는 순간 소멸 (개발노트 §3.1). 반드시 이 진단 도구로 볼 것
- aria-label 정규식 작성 시 마지막 행의 `"마지막 행"` 토큰, 마지막 열의 `"마지막 열"`
  토큰을 잊지 말 것 (`(?:마지막 행 )?` 패턴)
