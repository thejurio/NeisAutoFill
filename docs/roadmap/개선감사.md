## 총평

완성도 높은 실사용 도구. 아키텍처·안전정책·테스트·문서화가 개인 프로젝트 기준 매우 탄탄하다.
리스크는 코드 품질이 아니라 **외부 의존·보안·테스트 사각**에 집중돼 있었고, 이번 세션에 최우선 항목을 대부분 조치했다.

| 영역 | 평가 | 근거 |
|---|---|---|
| 아키텍처 | ★★★★★ | Core 의존성 0, 단방향 의존, 계층 경계 명확 |
| 안전 설계 | ★★★★★ | 자동 저장 억제·화면 실측 매칭·재검증·척도 화이트리스트 |
| 테스트/문서 | ★★★★☆ | 순수 로직 107개 커버 · 문서 체계화. Automation/App은 여전히 얇음 |
| 보안/개인정보 | ★★★★☆ | (조치 후) HMAC 인증+레이트리밋+로그 익명화. 공개 바이너리 시크릿 추출은 상한으로 방어 |
| 운영 견고성 | ★★★☆☆ | 나이스 DOM·단일 GAS 의존은 남음. 완화책은 갖춤 |

## 문제점 · 위험 (P)

| # | 항목 | 심각도 | 상태 | 조치/비고 |
|---|---|---|---|---|
| P1 | 무인증 AI 엔드포인트 — 쿼터 남용 가능 | 심각 | ✅ | V2 HMAC 서명 인증(서명·타임스탬프·nonce 리플레이) + 서버 레이트리밋. 시크릿 gitignore 분리. 라이브 검증 (v1.4.2) |
| P2 | 생성 실패 로그에 학생 실명 전송 | 심각 | ✅ | 실패 로그를 과목명만(`"수학 생성"`)으로 변경, 학생 실명 제거 |
| P3 | 나이스 DOM 의존 — 구조적 취약 | 심각 | ✅ | 원격 셀렉터(F14) 완화 — 개편 시 GAS 만 갱신해 앱 재배포 없이 대응. 잘못된 값은 검증·무시 |
| P4 | 가장 위험한 계층(Automation/App)에 테스트 0 | 중간 | 🔄 | BatchUploadRunner를 Automation 이동+가짜엔진 테스트 6개(F3). 나머지는 점진 |

| P6 | 자동업데이트 무결성 검증 없음 | 중간 | ✅ | 릴리스에 SHA-256 에셋 + 적용 전 해시 대조(불일치 중단). 하위호환 (v1.4.5 예정) |
| P7 | 업데이트 파일 교체 — 삭제 파일 잔류 | 낮음 | ✅ | robocopy /MIR 미러링으로 옛 파일 정리(빈 소스 가드 포함) |

## 잘된 점 (유지)

- **G1 깨끗한 5계층 분리** — 단방향 의존, Core 의존성 0
- **G2 안전 정책 내장** — 자동 저장 억제·화면 실측 매칭·재검증·화이트리스트
- **G3 하이브리드 매칭 + 확인 창** — 문제 있을 때만 사용자 확정
- **G4 회귀를 테스트로 덮은 리팩토링** — 실사용 버그 지점을 Core로 내려 고정
- **G5 데이터 유실 방지** — 작업공간 자동저장 + 즉시 영속화
- **G6 2단계 AI 파싱 + 범용 프롬프트** — 대형 문서 잘림 방지, 키 로테이션
- **G7 CDP attach + 실측 기반 배포 결정** — 계정 무위험, 근거 있는 선택
