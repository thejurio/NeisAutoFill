## 1. 이 프로그램이 하는 일

초등교사의 학기말 성적 처리를 처음부터 끝까지 지원. **담임**(우리 반 전과목)과 **전담**(여러 반의 내 과목) 두 모드.

```
[자료]      평가계획: PDF/HWP를 AI 인식(과목/학년·과목 선택 창) 또는 인앱 편집·엑셀
            명단: 인앱 붙여넣기.  담임=반 1개 / 전담=반별 명단 + 학년별 계획(반들이 공유)
[성적]      인앱 표 편집 (드래그+일괄버튼, ` 전체선택, 1~9 등급, Ctrl+Z/V) — 자동 저장
[AI 서술문] 백그라운드 생성 큐 (전과목/이 반, 품질 도우미, 엑셀 왕복)
[나이스]    등급→「교과별 평가」, 서술문→「학기말 종합의견」
            버튼 하나로: 화면 이동(어디서든) → 학년·반·과목 맞춤 → 조회 → 입력 → (배치는 저장)
            담임 배치 = 과목 순회 / 전담 배치 = 반 순회.  결과 대시보드 + 실패만 재시도
```

**핵심 안전 정책** (전 기능 공통)
- 나이스 [저장]: **단건 입력은 자동 저장 안 함** — 최종 확정은 사람이.
  **배치(전과목/전체반)만** 시작 시 동의 하에 "검증 통과 → 자동 [저장] → 다음"(A안).
  검증 실패는 저장하지 않고 그 자리에서 중단·리포트. "변경 없음"(저장버튼 비활성)은 **저장 생략 성공**으로 통과.
- 화면에 실제로 그려진 학생 이름·번호를 읽어 매칭 (추측 입력 없음).
  불일치는 확인 창(과목→학생→영역 3단계)에서 사용자가 결정. 배치 중 이름 매핑은 **1회만** 묻고 캐시 재사용('입력 안 함' 포함).
- 로그인 자동화 안 함 — 사용자가 로그인한 Edge에 CDP attach. 로그인 안 됐으면 상태칩·메시지로만 안내.
- 입력 후 값 재검증 + 1회 재시도. 잔여 알림 모달은 상호작용 직전마다 자동으로 닫음(DismissAlerts).

**UX 원칙** (사용자가 세션 중 확립 — 위반 금지)
- 팝업 최소화: 시작 확인창 금지(저장 안 하는 동작), 최상 시나리오 팝업 0개. 앱이 풀 수 있으면 조용히, 못 푸는 것(로그인)만 메시지.
- 카운트는 학생(명) 단위 표시. 로그는 중계식 금지("~하고 있어요…" 한 줄, 막힌 단계만 알림).
- 결과는 대시보드 하나로 통일(단건도 배치와 동일). 버튼: 왼쪽=선택·관리(추가가 제거 앞), 오른쪽=실행.
- API 키는 GAS 시트에서만 관리(입력 UI 금지). 사용 로그는 항상 서버로(옵트아웃 없음 — 안정화까지).

## 4. 담임/전담 두 모드 — 데이터 흐름과 분기 지점

### 4.1 축의 차이 (핵심 설계)
```
담임: 반 고정(자기 반) · 탭 = 과목      → 배치 = 과목 순회 (러너가 과목 콤보 전환)
전담: 과목 골라 [이동] · 탭 = 반        → 배치 = 반 순회   (runClass가 이동·전환, 러너 switchSubjects:false)
```

### 4.2 저장 위치 지도
```
문서\NeisAutoFill\                      ← 작업공간 (사용자 소유 엑셀)
 ├ 성적.xlsx 평가계획서.xlsx 서술문.xlsx     담임 (v1.2.x부터 그대로 — 무영향 보장)
 └ 전담\
    ├ 명단\{학년}-{반}.xlsx                반별 명단
    ├ 평가계획\{학년}학년.xlsx             학년별 계획(과목=시트, 담임 포맷 재사용)
    └ 작업\{학년}-{반}_{과목}\성적.xlsx     조합별 성적
%AppData%\NeisAutoFill\                 ← 앱 상태
 ├ settings.json(GeneratorOptions·LastRunVersion) scales.json state.json profile.json
 ├ narratives.json                        담임 서술문
 └ profiles\{학년}-{반}\narratives.json    전담 반별 서술문 (NarrativeStore.SwitchTo로 전환)
```

### 4.3 분기 지점 전수 (IsSubjectMode — App에 26곳)
리팩토링 시 이 목록이 회귀 방지 기준. 대부분 "표시/라벨/데이터 소스" 3종:
- **MainViewModel**: 생성자(InitSubjectAxis vs RestoreLastFiles) · 툴바(과목콤보+이동 vs 없음) · RunOne/RunAllLabel ·
  RunAllSubjects 분기(RunAllClasses) · RunSubjectCore(subjectModeClass 있으면 학년반과목+조회) · AutoSave(반별 파일) ·
  ConfirmSaveIfDirty · OnSubjectTabChanged(narratives 전환) · RefreshCriteriaPanel(전담 계획 소스) · OpenPlanEditor(store·initial) · OpenGenerator(SubjectModeGen·FocusUnit)
- **GeneratorViewModel**: _sm 유무로 Sheets()/Plans() 소스 전환 · 선택기 표시 · 라벨 · UploadAll 분기 · UploadCore ctx(축 선택 vs 과목만)
- **PlanEditorViewModel**: _subjectStore 유무로 반/학년 축 vs 단일 · import 분기(ImportUnitsAsync vs ImportAsync)
- **SubjectViewModel**: TabLabel/OwnerClass (담임은 기본값이라 무영향)

## 5. 나이스 연동 — 실측 확정 지식 (재발견 비용 큼, 삭제 금지)

### 5.1 화면 판별 (DetectStatusAsync)
| 신호 | 판정 |
|---|---|
| 문서 title `로그인` / `로그아웃` / URL `Logout` | LoggedOut (인증서 방식 — **비밀번호칸 없음**, 과거 추정 오판) |
| `div.app-tit` 텍스트 = `교과평가`/`학기말종합의견`/`교과학습발달상황` | 화면 종류 (그리드는 **조회 후에야 생김** — 판별·도착 확인에 그리드 요구 금지) |

### 5.2 메뉴 이동 (NavigateToAsync)
- 경로: **학급담임 → 학생평가 → {교과평가·학기말종합의견·교과학습발달상황}** (+교과평가만 하위 [교과별 평가] 탭)
- 상단 네비 `학급담임 0단계 메뉴항목`은 **어느 화면에서도 보임** → 어디서든 출발 가능
- 메뉴 텍스트에 `0단계 메뉴항목`·`2단계`·`3단계` 접미사 → **포함 매칭**(Exact 금지), a/menuitem/tab 우선 클릭+렌더 폴링
- 도착 확인 = 제목(app-tit)만. 이후 **조회는 호출부가 반드시 누름**(QueryAsync)

### 5.3 조회조건 콤보 (실측 함정 3개)
| 화면 | 라벨 상태 | 대응 |
|---|---|---|
| 교과별 평가 | `학년,2026`(=학년도 오라벨) + `학년,5`(진짜) | FindQueryCombo prefer=값 1~6 |
| 종합의견/세특 | **전부 `학기,…`로 깨짐** (`학기,5`=학년, `학기,1`=반, `학기,국어`=교과) | ClassifyNarrativeAxis — 값 성질(4자리=연도, 비숫자=교과, 잔여 숫자 순서=학기·학년·반) |
| 콤보 팝업 | 가상 렌더 — 스크롤 위치 유지됨 | 열면 **맨위 리셋 + 150ms 재렌더 대기** 후 스캔 (안 하면 위쪽 옵션 누락) |

### 5.4 버튼·그리드·알림
- [조회] 이름 매칭: **정확 일치 우선** — `조회 숨김`(접기 토글) 오클릭 방지. `숨김/펼` 포함은 폴백에서도 제외
- 그리드: 성적=colcount 8, 종합의견=**5** → FindGrid는 8 우선+아무 CLX 그리드 폴백. 서술문 입력기(NarrativeWriter)는 원래 5 기준
- 저장: 변경 없으면 버튼에 `cl-disabled` → **저장 생략 성공** 처리. 저장 후 대화상자 최대 2개(첫 4s·둘째 1.5s)
- **알림 모달**("변경된 내용이 없습니다" 등)이 남으면 이후 클릭 전부 차단 → 모든 상호작용 직전 `DismissAlertsAsync`
- 성능: 콤보 라벨은 **JS 일괄 1회**로 읽기 (행마다 등급 콤보가 있어 개별 RPC는 폴링당 수 초)
- 기존 확정(v1.3.x): 마지막 행 aria `"마지막 행"` 토큰, CLX `reveal(row)`, div.cl-textarea 동적 편집

## 7. 리팩토링 관찰 (다음 리팩토링 지시의 재료)

### 7.1 크기·책임 과다 (우선순위 순)
1. **MainViewModel 1,431줄** — 최소 6가지 책임 혼재: ①연결/상태 루프 ②담임 작업공간 ③전담 축(탭·이동·반별 저장) ④단건 입력 오케스트레이션(이동·축·조회·매칭·대시보드) ⑤배치 2종 ⑥창 열기 팩토리(자료준비·서술문의 클로저 조립). → 후보 분리: `NeisSessionController`(연결·상태·이동), `GradeUploadOrchestrator`(단건+배치, 담임/전담 전략), `SubjectModeAxis`(전담 축), 창 팩토리.
2. **GeneratorViewModel 1,154줄** — 생성 큐 UI + 전담 선택기 + 업로드 오케스트레이션 + 품질 + 엑셀 왕복. 업로드 부분이 MainViewModel의 성적 업로드와 **구조 중복**(사전점검→이동→축→조회→실행→대시보드) → 공용 오케스트레이터로 합칠 1순위.
3. **NeisEngine 856줄** — 판별/이동/콤보축/조회저장/등급입력이 한 클래스. RunSubjectAsync(등급 입력 본체)를 NarrativeWriter처럼 `GradeWriter`로 분리하면 대칭 맞음.

### 7.2 구조적 중복·냄새
- **업로드 파이프라인 4벌**(담임 성적/전담 성적/담임 서술문/전담 서술문)이 같은 골격을 각자 구현 — "대상 목록 + 대상별 준비(이동·축·조회) + 실행기 + 저장 정책"을 전략으로 추출하면 러너 하나로 수렴.
- **대시보드 Outcome 조립**이 Main/Generator에 각각(라벨·상태·msg switch) — 헬퍼 1곳으로.
- **_batchNameMap 캐시**가 Main/Generator에 별도 존재(같은 개념) — 매칭 세션 객체로.
- `SubjectAssignment.cs`(구 모델)·`TryGoToEvaluationAsync`(NavigateTo 위임 껍데기) — 제거 후보.
- MainViewModel 안 `Fail`/UnitPath 등 전담 헬퍼가 클래스 하단에 산재.
- 창 열기(OpenPlanEditor/OpenGenerator)의 **클로저 주입이 비대** — SubjectModeGen은 좋은 패턴이니 자료준비도 동일 컨텍스트 객체로.

### 7.3 리팩토링 시 지켜야 할 불변 조건
