## 1. 이 프로그램이 하는 일

초등교사의 학기말 성적 처리를 처음부터 끝까지 지원하는 도구.
엑셀 없이 프로그램 안에서 전 과정 진행 가능 (엑셀 왕복도 계속 지원).

```
[평가계획]   PDF/HWP/HWPX 를 AI 가 인식해 자동 입력 (이지에듀·스쿨마스터 등 양식 무관)
     │       또는 인앱 편집·엑셀 불러오기
[학생 명단]  인앱 입력 (엑셀 복사 → 한 번에 붙여넣기, 번호 자동)
     │       → 저장하면 성적표 자동 생성·동기화
[성적 채점]  인앱 편집 (다중 셀 복붙·일괄 지정·Ctrl+Z) — 작업공간 엑셀에 자동 저장
     │
[AI 서술문]  백그라운드 생성 큐 (전과목 일괄, Gemini·GAS 키 로테이션)
     │
[나이스 입력]
     ├ 등급    「교과별 평가」      ← 실기기 검증 완료
     ├ 서술문  「학기말 종합의견」   ← 실기기 검증 완료
     └ 🚀 전과목 자동: 과목 전환→입력→검증→저장 반복 (A안) ← 실기기 검증 완료
```

**핵심 안전 정책** (전 기능 공통)
- 나이스 [저장]: **단일 과목 입력은 자동 저장하지 않는다** — 최종 확정은 사람이.
  단, **전과목 자동 입력 모드에서만** 사용자가 시작 시 명시 동의하면
  "과목별 검증 통과 → 자동 [저장] → 다음 과목" 으로 진행 (2026-07-17 A안 결정).
  검증 실패 과목은 저장하지 않고 그 자리에서 중단·리포트.
- 화면에 실제로 그려진 학생 이름·번호를 읽어 매칭 (추측 입력 없음).
  불일치는 입력 전 매칭 미리보기 창에서 사용자가 직접 매핑·제외를 결정.
- 로그인 자동화 안 함 — 사용자가 로그인한 Edge에 CDP로 attach
- 입력 후 값 재검증 + 1회 재시도, 실패는 사유와 함께 리포트

### 개발 중 규명한 핵심 사실
- **§8 "마지막 행 미렌더링"의 진짜 원인**: 개발노트의 가상스크롤 가설은 틀렸음.
  실기기 DOM 덤프 결과, 마지막 행만 aria-label에 `"마지막 행"` 토큰이 추가됨
  (`"17행 마지막 행 성명 …"`). 정규식이 이를 못 읽어 누락됐던 것. → 정규식에 `(?:마지막 행 )?` 추가로 해결.
- **서술문 화면 편집 컨트롤**: 네이티브 `<textarea>`가 아니라 CLX 커스텀 `div.cl-textarea`.
  클릭하면 편집 모드로 진입해 실제 textarea가 동적 생성됨 (콤보 팝업과 같은 패턴).
- **CLX 공식 스크롤 API**: `cpr.core.Platform.INSTANCE.getAllRunningAppInstances()` 로
  그리드 컨트롤을 찾아 `reveal(rowIndex)` 호출 가능 (누락 행 복구 폴백으로 사용).
- **API 키는 GAS 시트에서 관리** — 프로그램에 키 입력 UI를 두지 않는다 (사용자 결정).
- **화면 요동 복구 금지** — 휠 연타/PageDown 등 화면을 떨리게 하는 복구는 제거 (사용자 결정).

## 3. 솔루션 구조 (5계층 + 테스트)

```
NeisAutoFill.sln
├─ src/NeisAutoFill.Core/         도메인 (외부 의존성 0 — 순수 C#, 테스트 대부분 여기)
├─ src/NeisAutoFill.Excel/        엑셀 읽기·쓰기 (ClosedXML)
├─ src/NeisAutoFill.Automation/   브라우저 엔진 (Playwright, CDP attach)
├─ src/NeisAutoFill.Generator/    AI 서술문 생성 (GAS 백엔드 호출)
├─ src/NeisAutoFill.App/          WPF UI (MVVM)
└─ tests/NeisAutoFill.Tests/      xUnit 75개
```

의존성 방향 (안쪽으로만): `App → {Automation, Excel, Generator} → Core`

## 5. 핵심 알고리즘

### 5.1 하이브리드 등급 매칭 (v1.2.0, StudentMatcher)
```
1. DetectNameProblem: 엑셀 영역명 중복 / 나이스 같은영역 중복 / 매칭 안 되는 영역명 검사
2. 문제 없음 → 이름 기반 (영역명으로 매칭, 순서 무관)  ← 기본·안전
3. 문제 있음 → 사용자에게 "순서대로 입력할까요?" 확인
     [예] → 순서 기반 (학생별 나이스 행 순서 ↔ 엑셀 영역 순서 위치 정렬)
             단, 영역 개수 다르면 FatalError → 중단 (오입력 방지)
     [아니오] → 중단
```

### 5.2 등급 입력 흐름 (NeisEngine.RunSubjectAsync)
```
attach 확인 → 화면 과목==대상 검증 → 그리드 탐색
→ RowMapBuilder(행 지도) → StudentMatcher(하이브리드) → todo
→ todo 순회: 행 가시화 → 콤보 열기 → 옵션 클릭 → 재검증 → 1회 재시도
→ 리포트 (성공/건너뜀/실패/누락) → 문제 있으면 팝업 안내
저장은 안 함
```

### 5.3 마지막 행 처리 (RowMapBuilder)
```
프록시 스크롤로 위→아래 훑으며 렌더된 행 스캔 (idx 키, 중복 스킵)
→ 누락 있으면 CLX reveal(idx) 폴백 → 재스캔
→ 진단: 행 지도 전체(rowindex→영역) + 파싱실패 원본 라벨 로그
(정규식이 '마지막 행' 토큰을 처리하므로 실제 누락은 거의 없음)
```

### 5.4 자동 연결 (MainViewModel.AutoConnectLoopAsync)
```
앱 시작부터 3초 주기 백그라운드 루프:
  미연결 → attach 조용히 시도 (이미 열린 브라우저 자동 포착)
  연결됨 → IsAliveAsync 확인 → 끊겼으면 재연결
UI 갱신은 Dispatcher로 마셜링, 엔진 호출은 백그라운드 (UI 안 멈춤)
```

### 5.5 입력 전 확인 (v1.3.0 — MatchAnalyzer + MatchPreviewWindow)
```
실행 → 엔진이 화면 행지도 파악 → resolveMatch 콜백
→ MatchAnalyzer.Analyze: 과목/학생/영역명/영역수 불일치 검사
→ Clean 이면 창 없이 ByName 진행
→ 문제 있으면 확인 창:
    과목 다름  → 동의 체크 필수
    학생 다름  → "이 자리에 누구의 성적" 드롭다운 (유사 이름 자동 추천 / 입력 안 함)
    영역 다름  → 화면 순서대로 한 줄씩 넣을 영역 선택
                 (이름 일치 자동 선택 → 남는 영역 순서 제안 → 남으면 '입력 안 함')
→ 확정 시 StudentMatcher 에 nameMap/OrderedExcelAreas 오버라이드로 반영
```

### 5.6 전과목 자동 업로드 (v1.3.0 — A안)
```
[🚀 전과목 입력] → 과목 체크리스트(자동 저장 동의 포함) → 과목마다:
  SelectSubjectAsync: 과목 콤보 선택 → [조회] → 화면 갱신 폴링
  → RunSubject/Narratives (확인 창 포함) → 실패 있으면 저장 없이 중단
  → 전부 통과 시 SaveScreenAsync: [저장] 클릭 → 확인/완료 대화상자 [확인]
→ 과목별 결과 요약 팝업. 단일 과목 입력은 기존대로 저장 안 함
※ 종합의견 화면은 콤보 라벨이 전부 "학기,"인 나이스 버그 → 값이 숫자가 아닌
  조회조건 콤보를 과목으로 인식하는 폴백 (2026-07-17 실측)
```

### 5.7 AI 평가계획 인식 (v1.3.0 — GasPlanImporter + code.gs)
```
파일 → 추출 (pdf 원문 / hwpx XML 텍스트 / hwp 한컴 COM→PDF)
→ 1차: listPlanSubjects — 문서의 과목명만 발견 (새 과목 자동 대응, 창체 제외)
→ 2차: 과목당 parsePlan(onlySubject) — 영역 누락 없이 완전 추출
  (범용 프롬프트 1개 — 이지에듀/스쿨마스터 등 양식 무관, 영역명 순수 규칙,
   중복 영역은 별개 행 유지 → 앱이 (2),(3) 번호. 모델 gemini-3.1-flash-lite, 404 폴백)
→ 편집 창 표에 채움 → 사용자 확인·수정 → 저장
```

### 5.8 백그라운드 서술문 생성 (v1.3.0 — GenerationQueue)
```
큐 투입(배치 로깅 1건) → 워커 2개 동시 실행
→ JobStarted("생성 중")/JobFinished 를 UI 스레드로 마셜링
→ 완료 즉시 narratives.json 저장 (창 닫아도·앱 꺼져도 완료분 보존)
→ 메인 하단 배지에 진행 상시 표시, [⛔ 중지] 가능
```

## 7. 빌드·배포

```
dotnet build                 # 전체 빌드
dotnet test                  # 75개 테스트
dotnet run --project src/NeisAutoFill.App
```

**릴리스** (버전 3곳 = csproj Version / git 태그 / 릴리스 제목)
```
1. src/NeisAutoFill.App/NeisAutoFill.App.csproj 의 <Version> 올리기
2. dotnet publish src/NeisAutoFill.App -c Release -r win-x64 --self-contained true -o publish/NeisAutoFill
   → 단일 exe(내부 압축)+.playwright+사용설명서.html (슬림 설정은 csproj 에 내장, v1.3.1)
3. Compress-Archive publish/NeisAutoFill/* publish/NeisAutoFill-win-x64.zip -Force   (약 97MB)
4. gh release create vX.Y.Z publish/NeisAutoFill-win-x64.zip --title "..." --notes-file notes.txt
→ 기존 사용자는 다음 실행 때 UpdateService가 감지 → 진행 창 → 재시작
  → 첫 실행에서 구버전 다중 DLL 잔재 자동 청소 (App.CleanupOldLayoutFiles)
```

**GAS (code.gs) 변경 시**: Apps Script 편집기에 로컬 code.gs 붙여넣기 → [배포 관리] →
기존 배포 수정 → **새 버전** (URL 유지). 새 배포로 만들면 URL 이 바뀌어 프로그램 설정도 바꿔야 함.
설명서 웹 서빙(?page=manual)은 다중 구글 계정 환경에서 오류가 있어 미사용 — 설명서는 앱 내장.

ref/(샘플 평가계획서·참고 코드), assets/ref·mockups, *.xlsx, *.zip — 개인정보·GAS 주소 보호
