# 원격 셀렉터로 나이스 개편에 대응하기 (F14)

나이스(NEIS)가 화면 구조(HTML/DOM)를 바꾸면, 프로그램이 화면 요소를 못 찾아 등급·서술문 입력이 실패한다.
**원격 셀렉터**는 그때 **앱을 새로 빌드·배포하지 않고**, GAS(`code.gs`)의 값 하나만 고쳐 재배포하면
모든 사용자가 다음 실행 때 자동으로 고쳐지게 하는 장치다.

> 이 문서는 **나중에 AI에게 그대로 시켜서 대응할 수 있도록** 아주 자세히 적었다.
> 실제 대응이 필요하면 이 문서 전체를 AI에게 주고 "나이스가 바뀌어서 OO가 안 된다, 이 문서대로 고쳐줘"라고 하면 된다.

---

## 0. 3줄 요약 (급할 때)

1. 무엇이 안 되는지 파악 → 아래 [§4 증상별 셀렉터 표](#4-증상별-어떤-셀렉터를-고칠지)에서 고칠 키를 찾는다.
2. 나이스 새 화면에서 올바른 셀렉터/정규식을 알아낸다 ([§5 새 값 알아내기](#5-새-셀렉터정규식-알아내기)).
3. `code.gs`의 `SELECTOR_OVERRIDES` 에 `"키": "새 값"` 을 넣고 **[배포 관리]→새 버전**으로 재배포 ([§6 적용](#6-gas에-넣고-재배포)). 끝.

## 1. 어떻게 동작하나 (전체 그림)

```
[나이스 화면 DOM]
       ▲ 셀렉터로 요소를 찾음
[앱: NeisSelectors]  ← 기본값(내장) 은 src/NeisAutoFill.Automation/NeisSelectors.cs
       ▲ 앱 시작 시 덮어씀(있으면)
[RemoteSelectorService]  ← src/NeisAutoFill.App/Services/RemoteSelectorService.cs
       │ 앱 시작 시 GAS 에 selectors 요청(서명)
       ▼
[GAS code.gs]  action:"selectors" → { ok, version, selectors: SELECTOR_OVERRIDES }
```

- **기본값**은 앱에 내장돼 있다(`NeisSelectors.cs`). 원격이 비어 있으면 이 값으로 동작 = **지금 상태**.
- 앱은 켜질 때마다 GAS 의 `selectors` 를 받아, 값이 있으면 기본값을 **덮어쓴다**(`NeisSelectors.ApplyRemote`).
- 잘못된 값(빈 문자열·컴파일 안 되는 정규식·캡처 그룹 없는 정규식)은 **앱이 검증해서 무시**하고 기본값을 유지한다.
  → 원격에 실수로 이상한 값을 넣어도 앱이 통째로 망가지지 않는다.
- 실패·오프라인·인증 실패면 그냥 기본값으로 계속 동작한다. **원격은 절대 앱을 막지 않는다.**

## 2. 지금 상태 (평상시)

`code.gs` 의 `SELECTOR_OVERRIDES` 는 **비어 있다(`{}`)**. 그래서 원격은 아무 효과가 없고 앱은 내장 기본값으로 동작한다.
→ 나이스가 바뀌기 전까지는 건드릴 필요가 없다. 바뀌었을 때만 아래대로 채운다.

```javascript
// code.gs 현재 (평상시)
const SELECTOR_OVERRIDES = {
  // 비어 있음 — 앱이 기본값 사용
};
```

## 3. 전체 셀렉터 키 목록 (무엇을 고칠 수 있나)

키 이름은 **C# `NeisSelectors` 프로퍼티명과 정확히 일치**해야 한다(대소문자 포함). 값은 전부 문자열.

### CSS 셀렉터 (화면 요소를 찾는 규칙)

| 키 | 무엇을 가리키나 | 현재 기본값 |
|---|---|---|
| `SubjectCombo` | 과목 콤보 (aria-label "교과, 국어") | `div[role='combobox'][aria-label^='교과,']` |
| `Grid` | 평가 그리드 (성취기준 탭과 구분: colcount=8) | `div.cl-grid[role='grid'][aria-colcount='8']` |
| `DataRow` | 그리드의 한 행 | `div.cl-grid-row[data-rowindex]` |
| `GridCell` | 그리드 셀 | `div[role='gridcell']` |
| `ComboInCell` | 셀 안의 등급(단계) 콤보 (셀 인덱스 6) | `div[role='gridcell'][data-cellindex='6'] [role='combobox']` |
| `OptionItem` | 콤보 열었을 때 뜨는 옵션 항목 | `.cl-global-aside .cl-combobox-item[role='option']` |
| `OptionText` | 옵션의 텍스트 요소 | `div.cl-text` |
| `VScroll` | 세로 프록시 스크롤바 (가상 스크롤) | `div.cl-grid-detail-band .cl-blank .cl-scrollbar` |
| `DetailBand` | 그리드 디테일 밴드 | `div.cl-grid-detail-band` |
| `AnyButton` | 버튼 일반 | `[role='button'], button` |
| `DialogButton` | 저장 확인·완료 대화상자 안의 버튼 | (여러 개, code 참고) |
| `DisabledClass` | 비활성 버튼에 붙는 CSS 클래스 | `cl-disabled` |

### 버튼·이름 (텍스트로 찾는 것)

| 키 | 의미 | 기본값 |
|---|---|---|
| `QueryButtonName` | [조회] 버튼의 이름(텍스트/aria-label 에 포함) | `조회` |
| `SaveButtonName` | [저장] 버튼의 이름 | `저장` |
| `DialogYesNames` | 대화상자 긍정 버튼 이름 후보. **쉼표로 구분** | `확인,예,저장` |

### 정규식 (aria-label 에서 값 뽑기)

행마다 `"3행 성명 김하늘"` 같은 aria-label 이 붙는데, 여기서 번호·성명·영역을 뽑는 정규식이다.
**반드시 캡처 그룹 `( )` 이 1개 이상 있어야 한다** (그 그룹이 실제로 뽑는 값). 없으면 앱이 무시한다.

| 키 | 무엇을 뽑나 | 기본값(정규식) |
|---|---|---|
| `NoRegex` | 번호 | `^\d+행 (?:마지막 행 )?반/번호 (.+?)(?:\s\|$)` |
| `NameRegex` | 성명 | `^\d+행 (?:마지막 행 )?성명 (\S+)` |
| `AreaRegex` | 평가 영역 | `^\d+행 (?:마지막 행 )?영역 (.+?)$` |

> ★ `(?:마지막 행 )?` 은 나이스가 **마지막 행에만** aria-label 에 "마지막 행"을 끼워넣는 실측 버그 대응이다.
> 새 나이스에서 이 토큰이 없어졌으면 그대로 둬도 되고(있어도 무해), 형식이 완전히 바뀌었으면 새로 짜야 한다.

## 4. 증상별 — 어떤 셀렉터를 고칠지

| 증상 | 원인 후보 | 고칠 키 |
|---|---|---|
| "과목 콤보를 찾지 못했습니다" | 과목 선택 콤보 못 찾음 | `SubjectCombo` |
| "평가 그리드를 찾지 못했습니다" | 성적 표 자체를 못 찾음 | `Grid`, `DataRow`, `GridCell` |
| 등급이 입력이 안 되거나 콤보가 안 열림 | 등급 콤보/옵션 못 찾음 | `ComboInCell`, `OptionItem`, `OptionText` |
| 스크롤해도 아래 행/과목이 안 잡힘 | 가상 스크롤 요소 변경 | `VScroll`, `DetailBand` |
| [조회]/[저장] 버튼을 못 누름 | 버튼 이름/구조 변경 | `QueryButtonName`, `SaveButtonName`, `AnyButton` |
| 저장 확인창의 [확인]을 못 누름 | 대화상자 구조/버튼명 변경 | `DialogButton`, `DialogYesNames` |
| 저장 버튼이 항상 비활성으로 보임 | 비활성 클래스명 변경 | `DisabledClass` |
| 학생 번호·이름·영역을 잘못 읽음(행 매칭 실패) | aria-label 형식 변경 | `NoRegex`, `NameRegex`, `AreaRegex` |

## 5. 새 셀렉터/정규식 알아내기

### 방법 A — 화면 진단 도구 (권장)
프로그램에 **화면 진단(DomInspector)** 이 내장돼 있다(현재 UI 버튼은 숨김, 코드는 살아있음 — [보관_진단_검증도구.md](../archive/보관_진단_검증도구.md) 참고).
이걸 켜면 현재 나이스 화면의 그리드 열/행 수, 각 셀의 aria-label, 콤보 라벨, 편집기 후보 등을
`%AppData%\NeisAutoFill\dom_inspect_*.txt` 로 덤프한다. 이 덤프를 보고 새 구조를 파악한다.

> 버튼을 다시 붙이는 법은 보관 문서에 있다. 진단 리포트(dom_inspect_*.txt)를 AI 에게 주면
> "이 구조면 셀렉터는 이렇게" 를 뽑아낼 수 있다.

### 방법 B — 브라우저 개발자도구 (F12)
나이스 화면에서 F12 → 요소 검사로, 안 되는 요소의 실제 태그·class·role·aria-label 을 직접 확인한다.
- CSS 셀렉터: 그 요소를 유일하게 집는 규칙을 만든다 (role·class·속성 조합).
- 정규식: 셀의 `aria-label` 실제 문자열을 보고, 값 부분을 `( )` 로 잡는 패턴을 만든다.

### CSS 셀렉터 작성 팁
- 되도록 **role·aria 속성** 기반으로 (class 는 자주 바뀜). 예: `div[role='gridcell'][data-cellindex='6']`.
- 콤마로 여러 후보를 나열하면 그중 하나만 맞아도 된다. 예: `.cl-dialog button, [role='dialog'] button`.

## 6. GAS 에 넣고 재배포

1. `code.gs` 를 Apps Script 편집기에서 연다 (어느 프로젝트인지는 [GAS_서버.md](GAS_서버.md) §1 참고).
2. `SELECTOR_OVERRIDES` 에 **바뀐 키만** 넣는다. 안 바뀐 키는 넣지 않는다(기본값 유지).
   ```javascript
   const SELECTOR_OVERRIDES = {
     "Grid": "div.cl-grid[role='grid'][aria-colcount='9']",   // 예: colcount 가 8→9 로 바뀐 경우
     "NameRegex": "^\\d+행 성명 (\\S+)",                        // 예: '마지막 행' 토큰이 없어진 경우
   };
   ```
   - **정규식은 JavaScript 문자열**이라 `\` 를 `\\` 로 이스케이프해야 한다 (`\d` → `\\d`, `\s` → `\\s`).
   - `DialogYesNames` 는 `"확인,예,저장"` 처럼 쉼표로 구분한 하나의 문자열.
3. `SELECTOR_VERSION` 을 1 올린다 (선택 — 로그로 추적할 때 편함).
4. **[배포] → [배포 관리] → (기존 배포) 편집 → 버전: 새 버전** 으로 배포. **URL 은 그대로 유지.**
   (새 배포를 만들면 URL 이 바뀌어 앱이 못 찾는다.)

> 앱은 재빌드·재릴리스하지 않는다. 사용자는 프로그램을 **다시 켜기만** 하면 새 셀렉터로 동작한다.

## 7. 적용 확인 (재배포 후)

PowerShell 로 selectors 액션이 새 값을 돌려주는지 확인한다 (시크릿·URL 은 로컬 값 사용):

```powershell
$url = "<GAS /exec URL>"      # 주소.txt
$secret = "<V2_SECRET>"       # GasSecret.cs / code.gs 의 V2_SECRET
$ts=[DateTimeOffset]::UtcNow.ToUnixTimeSeconds().ToString(); $nonce=[guid]::NewGuid().ToString("N")
$h=[System.Security.Cryptography.HMACSHA256]::new([Text.Encoding]::UTF8.GetBytes($secret))
$sig=([BitConverter]::ToString($h.ComputeHash([Text.Encoding]::UTF8.GetBytes("selectors|$ts|$nonce"))) -replace '-','').ToLower()
$body=@{action="selectors";authVersion="2";timestamp=$ts;nonce=$nonce;signature=$sig;clientName="verify"}|ConvertTo-Json -Compress
Invoke-RestMethod -Uri $url -Method Post -ContentType 'application/json' -Body ([Text.Encoding]::UTF8.GetBytes($body))
```

- `selectors` 필드에 방금 넣은 값이 보이면 서버는 정상.
- 그다음 **프로그램을 켜서** 실제로 안 되던 동작(등급/서술문 입력)이 되는지 나이스에서 확인한다.
  (앱은 잘못된 값을 무시하므로, 안 되면 값이 틀린 것 → §5 로 돌아가 값을 다시 잡는다.)

## 8. 검증 규칙 (앱이 원격 값을 어떻게 걸러내나)

`NeisSelectors.ApplyRemote` 가 다음을 **무시**한다 (무시된 키는 기본값 유지):
- **빈 문자열/공백**만 있는 값
- **모르는 키** (오타 포함 — 그래서 키 이름을 정확히 써야 한다)
- **컴파일 안 되는 정규식** (예: `([` 처럼 문법 오류)
- **캡처 그룹이 없는 정규식** (값을 뽑을 그룹 `( )` 이 없으면 무의미하므로 거부)
- `DialogYesNames` 는 쉼표로 나눈 뒤 항목이 하나도 없으면 무시

즉 **원격에 실수해도 앱은 안 깨지고 기본값으로 계속 동작**한다. 대신 "고쳤는데 여전히 안 됨"이면
값이 검증에서 걸러졌을 수 있으니 위 규칙(특히 정규식 이스케이프·캡처 그룹)을 다시 본다.

## 9. AI 에게 시킬 때 (프롬프트 예시)

> "나이스가 개편돼서 [등급 입력 / 서술문 입력 / 조회·저장 버튼 …]이 안 된다.
> `docs/maintenance/원격_셀렉터.md` 를 읽고, 아래 화면 진단 덤프(또는 F12 로 확인한 DOM)를 참고해서
> 고쳐야 할 셀렉터 키와 새 값을 정하고, `code.gs` 의 `SELECTOR_OVERRIDES` 에 넣을 코드를 만들어줘.
> 정규식은 JS 문자열 이스케이프(`\\`)와 캡처 그룹을 꼭 지켜줘. 그리고 재배포·확인 절차도 알려줘."

덤프 파일: `%AppData%\NeisAutoFill\dom_inspect_*.txt` (화면 진단 실행 시) 또는 F12 요소 검사 결과.
